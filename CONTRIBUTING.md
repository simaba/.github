# Contributing Guidelines

Thank you for your interest in contributing to this project! These repositories
focus on AI governance, release readiness, and responsible AI deployment for
regulated industries. All contributions that advance these goals are welcome.

## What We Welcome

- **New use cases** — real-world scenarios where these frameworks apply
- **Documentation improvements** — clearer explanations, better examples, fixed typos
- **Framework extensions** — additional checklists, risk categories, governance patterns
- **Case studies** — anonymized examples of AI governance challenges and solutions
- **Tooling** — scripts or utilities that complement the existing frameworks
- **Translations** — making these resources accessible to non-English speakers

## How to Contribute

### 1. Open an Issue First

Before opening a pull request, please open an issue to describe:
- What you want to add or change
- Why it would benefit practitioners in regulated industries
- Any relevant context (industry, regulation, use case)

This avoids duplicated effort and ensures contributions align with the project vision.

### 2. Fork and Branch

```bash
git clone https://github.com/simaba/<repo-name>
git checkout -b feature/your-descriptive-branch-name
```

### 3. Make Your Changes

- Follow the existing document structure and formatting conventions
- For Markdown files: use ATX headings (`#`), fenced code blocks, and tables where appropriate
- For YAML configs: maintain the existing schema and add comments explaining new fields
- Include at least one practical example for any new framework element

### 4. Open a Pull Request

- Describe what you changed and why
- Reference the issue number (`Closes #123`)
- Be patient — reviews may take a few days
